Chillennium

Chillennium is an annual student-led 48 hour game design competition. A theme is provided by the organization and then teams of up to 4 people work together to design a game within the allotted time and in accordance with the given theme. The games are graded by pre-selected judges after the game jam is finished. Awards are then given out based on the judges' scores. I have participated every year since 2016.

In Chillennium 2016, I was in a group with 3 other people I had never met before- two artists and an animator. I was the sole programmer for this game. This was my first Chillennium and my first time using Unity and C#. The theme for this game jam was "foofaraw". We created a 2D platformer with 3 levels where the main character was a princess that yielded a pistol that she used to defeat enemies in her way. Since I was the only programmer, I constructed the game physics and rules.

In Chillennium 2017, I was in a group with one friend and two strangers- the two strangers were artists, and my friend and I were both programmers. I learned from Chillennium 2016 that an iterative level-based game was very hard to make in 48 hours, so I went into this Chillennium with the idea to create a shorter game that could be replayed multiple times. The theme for this game jam was "role reversal". The game we created was a two-player 2D game where every 30 seconds, the controls for each player switches, causing them to control the other player's character. The goal of the game is to fill up a wheel barrel in the center of the level with a certain amount of candy. However, the two players are also competing to see who can contribute more candy to the pile. Whoever contributes more candy to the pile once the candy quota is met is the winner of the game. I worked on the game physics, rules, and animations.

Download the game here

In Chillennium 2018, I was in a group with one friend and one stranger- all of us were programmers, but one of us was assigned to be an artist. The theme for this game jam was "transformation". The game we created was a 2D puzzle-platformer where the player is meant to control a construction worker to reach the flag at the end of the level. In each level, the player is responsible for placing a limited number of platforms in order to cross the gap to reach the goal on the other side. I worked on the game physics, rules, the level design of each stage, and animations. This game was a Chillennium Award Finalist.

Hackathons

TAMUhack is a 24 hour hackathon hosted by Texas A&M that takes place every year. Each contestant has the option to join a team or work solo.

I participated in the Fall 2016 TAMUhack individually. I practiced my skills with Python and created a simple statistical analysis program that, when given a set of data, can return the mean, median, mode, range, standard deviation, and various other data.

I participated in the Spring 2019 TAMUhack in a group of 5 programmers. We worked on a browser application using Node.js. Our goal was to create an online multiplayer game where users would all be presented with a tweet from Twitter and then be prompted to give the tweet a hashtag. All players would vote on their favorite hashtag, and the user with the most votes at the end of the round would be the winner. I worked on the front end of the website with one other person and we used HTML, CSS, and JavaScript.

HowdyHack is a hackathon that is hosted by Aggie Coding Club, and the rules are the same as TAMUhack. I worked with one other person when I participated in the Fall 2018 HowdyHack. Together, we brainstormed an idea to create a chatbot that professors could use to provide answers to their students for frequently asked questions. For example, if a student wants to know the date of the next exam, rather than sending an email to the professor, the idea was to make a chatbot that a professor can customize and link on their teacher website that will answer the student's question. We made a prototype using Microsoft Azure, which makes use of natural language processing and machine learning to match a set of questions to a set of answers.

Aggie Coding Club

Aggie Coding Club is an organization at Texas A&M University that promotes teamwork on extracurricular projects. Every semester, we have project groups meet during the week to develop some kind of software as a team. The project ideas come directly from club members, and the club officers select a group of project managers to lead a project group. I have been a member of Aggie Coding Club since 2016.

I was selected to be a project manager by the officers for two consecutive semesters. The first team I led was creating an app to help factory workers voice their complaints with working conditions anonymously over SMS messaging. The other team I led wanted to make an app for resident advisors in on-campus dorms. The app would contain an easy lookup system for rules and emergency protocols that the resident advisors would be able to access much quicker than their current method of flipping through a large stack of papers.

I was also a team member in the construction of the Aggie Coding Club website. This was my first experience with web development. We used React.js to edit the existing website. I mainly edited and organized the existing HTML and CSS files to make future development easier and more streamlined.

The final project that I worked on with a team of 4 artists and 2 programmers was a VR game called Reflector VR. In this game, the player is put in the center of a testing facility where turrets spawn all around the player in a circle and bombard them with lasers fired from all directions. The player has a shield, which reflects the lasers back to the turrets, and a gun that can fire small lasers in quick succession. There are 3 levels and the goal is to destroy all the turrets and survive until the end of the game.

Computer Organization

In this course, we were instructed to build all of the hardware and software components that make a computer function. I was exposed to many low-level programming concepts in this course including assembly and machine language.

I built a fully functional computer from first principles and built and successfully ran rudimentary graphics on it. I also learned how to construct basic logic gates, as well as arithmetic, logic, and sequential chips. I crafted a simple processor architecture through an intuitive machine language supported with an implementation of higher abstractions including a virtual machine, compiler, and a Java-based high-level object oriented language called Jack.

As an end-of-course project, I created a game using the computer we had constructed.

Computer Systems

I constructed and learned about several different components that make up a UNIX operating system.

I used C++ in conjunction with a UNIX library to create a memory management system, a bash shell, and several inter-process communication methods- shared memory, named and unnamed pipes, and message queues. I also utilized threads and semaphores to implement client-server relationships to allow for data transfer.

Operating Systems

Throughout the duration of this course, I created a simple operating system that can be booted from an image file.

The operating system I made was written in C++. The components that I made included a frame manager and page manager to implement virtual memory, threads and a thread scheduler for concurrently running programs, support for multiprogramming with I/O bound threads, and a custom file system that I had to design and implement.

Computer and Network Security

This course provided a brief overview of various computer vulnerabilities, cyber threats, and how to defend against them.

I learned about buffer overflow vulnerabilities, public key and private key cryptography, hash functions, TCP/IP vulnerabilities and defenses, firewalls, and the network stack. To understand threats to security, this class provided a digital environment where I executed various attacks and learned what safeguards exist to prevent the attacks from succeeding.
